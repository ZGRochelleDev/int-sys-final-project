Project outline:
    My proposed improvement enhances TS-RAG by introducing an MMR-based retrieval mechanism. The original method retrieves time-series segments based solely on embedding similarity, which can lead to redundant results that capture the same pattern. To overcome this, I modify the retrieval step to first obtain a larger pool of candidate segments and then apply MMR to select a subset that balances relevance and diversity.

    This change ensures that the final retrieved set provides broader coverage of useful historical patterns, rather than repeated variations of the same pattern. The idea builds on established concepts from information retrieval and aligns with principles from informed search and unsupervised learning. By incorporating diversity into the retrieval process, the model is provided with richer contextual information, which can improve forecasting performance and robustness.

    Overall, this improvement sits at the intersection of retrieval-augmented generation (RAG), information retrieval, and time-series similarity search, making it a natural and well-supported extension of the TS-RAG framework.

Weakness: TS-RAG retrieves by embedding similarity only
    TS-RAG retrieves the top-k similar time-series segments from a knowledge database and uses the ARM module to fuse these retrieved patterns with the forecasting model. While this improves zero-shot forecasting and interpretability, the current implementation relies on FAISS IndexFlatL2, meaning retrieval is based purely on Euclidean nearest-neighbor search in embedding space. As a result, the retrieved top-k segments are often highly similar to each other, leading to redundancy and limited diversity in the contextual information provided to the model.

Proposed Improvement: MMR-Based Diversity-Aware Retrieval
    To address this limitation, I propose replacing the standard nearest-neighbor retrieval step with a Maximal Marginal Relevance (MMR)-based retrieval strategy. Instead of selecting the top-k closest segments solely based on similarity, MMR balances two objectives:

     - Relevance to the query (similarity to the input time series)
     - Diversity among selected segments (dissimilarity to already selected results)

    This approach first retrieves a larger candidate pool and then iteratively selects the final top-k segments by maximizing a combined score that penalizes redundancy. This ensures that the retrieved set contains both highly relevant and diverse time-series patterns.

Connection to Course Topics
    1. Informed Search: Retrieval is guided by a heuristic scoring function. MMR extends this by incorporating both similarity (relevance) and diversity into the search objective.
    2. Unsupervised Learning / Clustering: MMR implicitly groups similar candidates and selects representative samples, similar to choosing cluster representatives.
    3. Feature Selection: The method selects the most informative subset of retrieved segments rather than using all nearest neighbors.
    4. Local Search: The algorithm iteratively refines the selected set of retrieved segments, improving it step-by-step based on a scoring function.
